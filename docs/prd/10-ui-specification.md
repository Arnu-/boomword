# UI规范与交互说明

## 1. 设计原则

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| 游戏化 | 界面充满趣味性，使用动效和反馈增强游戏感 |
| 简洁 | 信息层级清晰，减少用户认知负担 |
| 一致性 | 统一的视觉语言和交互模式 |
| 可访问性 | 考虑不同用户的使用需求 |
| 响应式 | 适配多种屏幕尺寸 |

### 1.2 设计关键词

- 活泼、轻松、有趣
- 学习与娱乐结合
- 成就感和正反馈
- 清新、明亮的色调

## 10. 动效规范

### 10.1 过渡动画

| 场景 | 时长 | 缓动函数 |
|------|------|----------|
| 按钮悬停 | 150ms | ease-out |
| 页面切换 | 300ms | ease-in-out |
| 弹窗出现 | 200ms | ease-out |
| 弹窗消失 | 150ms | ease-in |
| 列表项出现 | 200ms | ease-out |

### 10.2 游戏动效

| 场景 | 效果 |
|------|------|
| 泡泡出现 | 从小到大缩放 + 淡入 |
| 泡泡漂浮 | 缓慢上下/左右浮动 |
| 泡泡高亮 | 添加发光边框，轻微放大 |
| 泡泡爆炸 | 缩放+透明度+粒子效果 |
| 泡泡抖动 | 左右快速摇摆 |
| 分数飘出 | 向上飘动+淡出 |
| 连击显示 | 放大+跳动 |

### 10.3 反馈动效

| 场景 | 效果 |
|------|------|
| 正确输入 | 绿色闪烁+震动 |
| 错误输入 | 红色闪烁+摇头 |
| 星级获得 | 依次亮起+旋转 |
| 升级 | 全屏光效+烟花 |
| 成就解锁 | 弹出+放大+发光 |

| ESC | 清空输入/暂停游戏 |
| 触摸输入 | 移动端弹出软键盘 |

### 12.3 表单交互

| 交互 | 说明 |
|------|------|
| 聚焦 | 边框颜色变为主色 |
| 失焦校验 | 离开时立即校验 |
| 错误提示 | 输入框下方红色提示 |
| 提交 | 按钮显示loading |

### 12.4 反馈交互

| 类型 | 方式 |
|------|------|
| 成功 | Toast提示+绿色图标 |
| 错误 | Toast提示+红色图标 |
| 加载 | 全屏/局部loading |
| 空状态 | 插画+文案+操作按钮 |
